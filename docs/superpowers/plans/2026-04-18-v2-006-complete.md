# V2-006 Complete Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Finish `V2-006` end-to-end by adding shift sessions, branch runtime policies, and workforce audit export on top of the existing attendance and cashier-session foundation.

**Architecture:** Extend the existing `workforce` authority boundary rather than introducing separate scheduling or policy subsystems. Shift sessions sit above attendance, branch runtime policy gates Store Desktop behavior, and workforce audit export extends the existing audit-event record rather than inventing a second report store.

**Tech Stack:** FastAPI, SQLAlchemy, Alembic, React, TypeScript, Vitest, pytest
